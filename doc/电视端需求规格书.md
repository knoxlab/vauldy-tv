# Vauldy 电视端客户端

> 需求规格说明书（SRS）

---

## 1. 项目概述

### 1.1 项目背景

Vauldy 是一款自托管的家庭数字保险箱，以 Go 后端 + React Web 前端提供影视、音乐、照片、文档等媒体的安全存储与在线浏览能力。客厅电视是家庭媒体消费的核心场景：用户期望在 **Android TV** 与 **Apple TV** 上以遥控器操作，大屏浏览家中 Vauldy 媒体库，享受与 Web / 移动端一致的内容访问体验。

为满足智能电视场景，需开发 **Vauldy 电视端客户端**（以下简称「电视客户端」），对接 Vauldy Media Server 现有 API，在电视大屏上提供聚焦式（10-foot UI）浏览与播放能力。

### 1.2 项目目标

基于 **react-native-tvos** 跨端框架，开发 Vauldy 电视端客户端，实现：

- 对接 **Vauldy Media Server** REST API（`/api/v1`）及流媒体端点，无需修改后端核心协议
- 一套代码适配 **Android TV** 与 **Apple TV (tvOS)**
- 支持媒体库浏览及 **视频、音乐、图片、文档** 在线预览/播放
- 遥控器 / D-pad **焦点导航** 优先，符合电视平台交互规范
- UI 风格与 **Vauldy Web** 保持一致（深色主题、品牌色）
- 独立代码仓库维护（[knoxmedia/Vauldy-TV](https://github.com/knoxmedia/Vauldy-TV)），通过 Git Submodule 嵌入主项目便于联调

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| Vauldy Media Server | 自托管媒体服务端，默认监听 `8200` 端口 |
| 电视客户端 | 本项目的 react-native-tvos 应用，运行于 Android TV / tvOS |
| react-native-tvos | React Native 的 TV 分支，支持 Apple TV 与 Android TV 构建 |
| 10-foot UI | 观看距离约 3 米的大屏界面设计规范 |
| Focus Engine | React Native TV 焦点系统，响应遥控器方向键导航 |
| 播放计划（Playback Plan） | `GET /media/:id/hls` 返回的 JSON，描述直链、HLS、DRM、JIT 等播放方式 |
| access_token | JWT；流媒体 URL 可通过 `?access_token=` 鉴权 |
| Leanback | Android TV 官方 UI 框架与设计规范 |

### 1.4 项目范围

**包含：**

- 终端用户（admin / user）的日常消费功能
- Android TV、Apple TV 双平台发布（`.apk` / App Store tvOS）
- 视频 HLS/直链播放、音乐播放、照片浏览、文档阅读

**不包含（首期）：**

- 管理后台（`adm` 路由：库管理、扫描、转码、用户管理等）
- 本地上传媒体至服务器
- 完整离线媒体缓存（可作二期）
- 演职人员模块（待后端 API 就绪）
- 手机端触控交互（电视端以遥控器为主）

### 3.2 技术选型

| 层次 | 技术 | 说明 |
|------|------|------|
| 框架 | **react-native-tvos** | 同时支持 tvOS 与 Android TV；与 RN 生态兼容 |
| 语言 | TypeScript | 复用 Web / 移动端 API 类型与 URL 工具 |
| 导航 | React Navigation 6+（TV 适配） | Stack + Tab；禁用不适用的手势返回 |
| 焦点 | RN 内置 `hasTVPreferredFocus`、`nextFocus*` | Android TV D-pad；tvOS Siri Remote |
| 状态 | Zustand | 认证、播放器、设置 |
| HTTP | Axios | 对齐 `web/src/api/client.ts` |
| 视频 | react-native-video（TV 分支） | ExoPlayer（Android TV）/ AVPlayer（tvOS） |
| 音频 | react-native-video 或 react-native-track-player | 电视端后台播放需求较低，一期可用视频组件播音频 |
| 图片 | RN Image / Fast Image | 大图渐进加载 |
| PDF | WebView + pdf.js 或 react-native-pdf | 遥控器翻页 |
| EPUB | WebView + epub.js | 与 Web / 移动端一致 |
| 安全存储 | react-native-keychain / EncryptedStorage | JWT 加密存储 |
| 国际化 | i18next | zh-CN、zh-TW、en、ja、ko |
| 构建 | Xcode（tvOS）+ Gradle（Android TV） | 可选 Expo TV 插件评估（二期） |

## 4. 功能需求

### 4.1 服务器连接与认证

#### 4.1.1 服务器配置

- 首次启动或设置中配置服务器地址（`http(s)://host:8200`）
- `GET /health` 连接测试
- 电视端输入：屏幕键盘（tvOS）/ 系统输入法（Android TV）
- 配置持久化（AsyncStorage / EncryptedStorage）
- 支持记住多个服务器（二期）

**验收标准：**
- 局域网 IP 可连接（如 `http://192.168.1.10:8200`）
- 连接失败给出明确提示

#### 4.1.2 用户登录

| API | 说明 |
|-----|------|
| POST `/api/v1/user/login` | 用户名密码 → JWT |
| GET `/api/v1/user/info` | 用户信息、权限 |
| GET `/api/v1/branding` | 应用名称、图标 |
| POST `/api/v1/user/logout` | 登出 |

- Token 安全存储
- 401 自动跳转登录
- 登录页支持遥控器焦点切换输入框

#### 4.1.3 权限与家长控制

- 遵循 `can_play`、`can_download`、`library_scope`、`parental_enabled`
- PIN 解锁：`POST /user/parental/unlock`
- 电视端 PIN 输入：大屏数字键盘 + 焦点导航

---

### 4.2 主界面与导航

#### 4.2.1 布局（10-foot UI）

- **侧边栏 / 顶栏导航**（对齐 Web MainNav，适配 TV 焦点）：
  - 首页、浏览、收藏、搜索、设置
- **内容区**：横向滚动货架（继续观看、最近添加、媒体库）
- **焦点指示**：选中项放大 / 描边高亮（`#00a4dc`）
- **安全边距**：距屏幕边缘 ≥ 48dp，避免 overscan 裁切

#### 4.2.2 遥控器映射

| 按键 | 功能 |
|------|------|
| ↑ ↓ ← → | 焦点移动 |
| OK / Select | 确认 / 播放 |
| Back | 返回上一级 |
| Play/Pause | 播放/暂停（播放页） |
| 快退/快进 | 视频 ±10s（播放页） |
| Menu（tvOS） | 弹出操作菜单 |

#### 4.2.3 首页

**展示内容：**
- 已授权媒体库卡片（横向滚动）
- **继续观看**（`GET /api/v1/user/history`）
- **最近添加**（`GET /api/v1/media?sort=created_desc`）

**验收标准：**
- 默认焦点落在第一个可交互元素
- 货架支持水平滚动且焦点跟随

---

### 4.3 媒体库浏览

与 Web / 移动端 API 对齐，按库类型展示：

| library.type | 浏览 UI |
|--------------|---------|
| movie / video | 海报网格（2:3），横向货架 |
| tv / anime | 剧集列表 → 季/集 |
| music | 专辑 / 艺术家 / 曲目货架 |
| photo | 时间线 / 分类网格 |
| document | 文件夹 / 格式分类列表 |

**API：**

- `GET /api/v1/library`
- `GET /api/v1/media`（筛选、排序、搜索）
- 剧集/音乐/照片/文档专用端点（同移动端规格书 4.3 节）

**展示：**

- 网格虚拟化（大库性能）
- 海报 URL 附加 `access_token`
- 列表 / 网格视图（二期）

### 4.4 视频播放

#### 4.4.1 播放流程

```
用户选中并确认播放
    ▼
GET /api/v1/media/:id/hls  → 播放计划
    ▼
构造播放 URL（hls_master / fallback / play）
    ▼
附加 ?access_token=
    ▼
全屏播放器（react-native-video）
    ▼
进度上报 POST /media/:id/progress
```

#### 4.4.2 播放器能力（一期）

| 功能 | 实现 |
|------|------|
| HLS 预转码 | 原生 demuxer |
| 直链 MP4/MKV | progressive 播放 |
| 断点续播 | 启动 seek 到历史进度 |
| 播放/暂停 | 遥控器 OK / Play键 |
| 快进/快退 | ±10s |
| 字幕 | 内嵌 / 外挂 VTT（二期完善） |
| 多音轨 | 播放菜单选择（二期） |

#### 4.4.3 DRM 说明

| DRM | 电视端支持 |
|-----|------------|
| HLS AES-128 | 一期可行（非 DRM 库优先） |
| Widevine | Android TV，二期 |
| FairPlay | tvOS，二期 |
| PowerDRM | 需专用解密，二期（参考 Web PowerPlayer） |

一期优先 **非 DRM 库** 与 **HLS 预转码 / 直链**；DRM 库提示使用 Web 端或标注「暂不支持」。

## 5. 接口集成规范

### 5.1 认证

- JSON API：`Authorization: Bearer <JWT>`
- 流媒体 / 图片：`?access_token=<JWT>`

### 5.2 URL 构建

复用 Web 端 `withAccessToken`、`mediaPosterSrc`、`mediaPlaySrc` 逻辑（`web/src/api/client.ts`），电视端需 `serverUrl` 前缀（同移动端 `mediaUrl.ts`）。

### 5.3 错误处理

| 状态 | 行为 |
|------|------|
| 401 | 清除 Token，跳转登录 |
| 403 | 权限提示（含家长控制） |
| 网络错误 | 重试 + 离线提示 |

## 6. 非功能需求

### 6.1 性能

| 指标 | 目标 |
|------|------|
| 冷启动 | ≤ 4s（已登录，TV 硬件） |
| 首页首屏 | ≤ 2s（局域网） |
| 视频起播 | ≤ 5s（HLS/直链） |
| 焦点响应 | ≤ 100ms |
| 内存 | 播放时 ≤ 300 MB |

### 6.2 兼容性

| 平台 | 最低版本 |
|------|----------|
| Android TV | Android 8.0（API 26），Leanback 启动器 |
| Apple TV | tvOS 15.0 |

### 6.3 安全

- Token 存系统安全存储
- HTTPS 支持；自签名证书需用户确认（内网）
- 禁止日志输出 Token

### 6.4 可用性

- 深色主题默认，与 Web 一致
- **所有可点击元素必须可聚焦**（`focusable={true}`）
- 最小可聚焦目标 48×48dp
- 高对比度焦点环
- 基础无障碍：VoiceOver / TalkBack 朗读标题
