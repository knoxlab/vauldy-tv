# Vauldy 电视端客户端

> 需求规格说明书（SRS）

---

## 1. 项目概述

### 1.1 项目背景

Vauldy 是一款自托管的家庭数字保险箱，以 Go 后端 + React Web 前端提供影视、音乐、照片、文档等媒体的安全存储与在线浏览能力。Web 端、移动端与桌面端已覆盖浏览器、手机/平板及 PC 场景，但家庭客厅中的 **大屏电视** 仍是核心消费场景：用户期望在 Android TV、Google TV、Apple TV 等设备上，用遥控器舒适地浏览片库、续播剧集、家庭相册放映与文档演示。

电视端交互与触控/mobile 差异显著：无精确指针、依赖 **焦点导航（Focus Navigation）** 与 **方向键/遥控器**，UI 需遵循 **10-foot 体验**（远距离可读、大字号、横向货架布局）。为满足客厅场景，需开发 **Vauldy 电视端客户端**（以下简称「电视客户端」），在 Android TV 与 Apple TV 上提供统一体验，并复用 Vauldy Media Server 现有 API，无需改动后端核心协议。

### 1.2 项目目标

基于 **react-native-tvos** 跨端框架，开发 Vauldy 电视端客户端，实现：

- 对接 **Vauldy Media Server** REST API（`/api/v1`）及流媒体端点
- 一套代码同时适配 **Android TV** 与 **Apple TV（tvOS）**
- 支持媒体库浏览及 **视频、音乐、图片、文档** 在线播放/预览
- 提供符合电视平台规范的 **焦点导航、遥控器映射、Leanback 式货架 UI**
- UI 风格与 **Vauldy Web / 移动端** 保持一致（深色主题、`#0f1419` / `#00a4dc` / `#ed6d00`），降低家庭用户学习成本
- 独立代码仓库维护（`knoxmedia/Vauldy-TV`），通过 Git Submodule 嵌入主项目便于联调

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| Vauldy Media Server | 自托管媒体服务端，默认监听 `8200` 端口，提供 API 与流媒体服务 |
| 电视客户端 | 本项目的 react-native-tvos 应用，运行于 Android TV / Apple TV |
| react-native-tvos | React Native 官方电视分支，支持 Android TV 与 tvOS 构建 |
| 10-foot UI | 面向 3 米观看距离的大屏界面设计规范，强调可读性与焦点可见性 |
| Focus / 焦点 | 当前遥控器选中的 UI 元素；仅焦点元素响应「确认」操作 |
| Leanback | Android TV 推荐的内容浏览模式：横向滚动货架 + 大卡片的沉浸式布局 |
| 播放计划（Playback Plan） | `GET /media/:id/hls` 返回的 JSON，描述直链、HLS、DRM、JIT 等播放方式 |
| access_token | JWT 令牌；流媒体资源 URL 可通过查询参数 `?access_token=` 鉴权 |
| JIT 转码 | 按需实时转码生成 HLS 流，无需预先打包 |
| DRM | 内容加密方案，包括 Widevine（Android TV）、FairPlay（tvOS）、PowerDRM、HLS AES-128 |
| Top Shelf | Apple TV 主屏顶部推荐位，展示继续观看等内容（tvOS 特性） |
| Android TV Leanback Launcher | Android TV 主屏，可注册内容 Provider 展示推荐行（二期） |

### 1.4 项目范围

**包含：**

- 终端用户（admin / user 角色）的客厅消费功能（浏览、播放、收藏、搜索、续播等）
- 与服务端已有 `/api/v1` API 的完整对接
- Android TV 与 Apple TV 双平台发布（`.apk` / App Store tvOS）
- 遥控器/游戏手柄方向键、确认键、播放/暂停等媒体键映射
- 视频 HLS/直链播放、音乐播放、照片幻灯片、文档阅读

**不包含（首期）：**

- 管理后台功能（库管理、扫描、转码任务、用户管理、系统选项等 `adm` 路由）
- 本地上传媒体文件至服务器
- 完整离线媒体缓存（可作三期）
- 演职人员（Person）模块（待后端 API 路由正式注册后再纳入）
- 电视端管理服务端配置（仅消费端；服务器地址在首次引导或设置中配置）
- 游戏手柄除基本导航外的复杂映射（三期可选）
- 直播/电视频道（非 Vauldy 核心能力）

## 2. 用户角色

| 角色 | 描述 | 电视端权限 |
|------|------|------------|
| 管理员（admin） | 服务端管理员账号 | 与 Web 端一致：可浏览、播放所有已授权库；**不**提供管理控制台 |
| 普通用户（user） | 家庭成员账号 | 按库级 ACL、文件夹权限、家长控制策略访问 |
| 未登录访客 | 未认证用户 | 仅可访问服务器配置页与登录页 |

**电视场景说明：**

- 客厅设备常为家庭共用账号登录，需支持 **登出** 与 **切换用户**（重新登录）
- 儿童模式依赖服务端 **家长控制** + PIN 解锁，电视端不提供绕过机制
- 管理员在电视上与 viewer 用户体验一致，不暴露后台入口

## 3. 系统架构

### 3.1 总体架构

```
┌─────────────────────────────────────────────────────────────────────┐
│              Vauldy TV (react-native-tvos App)                       │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌────────────┐ │
│  │  UI 层       │ │ 焦点/遥控器   │ │  API 客户端   │ │ 原生播放器  │ │
│  │  (RN + TS)   │ │  Focus Engine│ │  (Axios)     │ │  Exo/AVPlayer│ │
│  └──────────────┘ └──────────────┘ └──────────────┘ └────────────┘ │
│         │                 │                  │              │        │
│         └─────────────────┴──────────────────┴──────────────┘        │
│                    Zustand 状态 · i18n · 安全存储                      │
└───────────────────────────────┬─────────────────────────────────────┘
                                │ HTTPS / HTTP
                                │ Authorization: Bearer <JWT>
                                │ ?access_token=<JWT> (流媒体/图片)
┌───────────────────────────────▼─────────────────────────────────────┐
│                 Vauldy Media Server (:8200)                          │
│  Gin API · SQLite · FFmpeg · DRM License · JIT Session               │
└─────────────────────────────────────────────────────────────────────┘
```

**数据流概要：**

1. 用户在电视端配置服务器地址并完成登录，JWT 存入平台安全存储
2. JSON API 经 Axios 携带 `Authorization: Bearer` 访问 `/api/v1`
3. 播放前请求 `GET /media/:id/hls` 获取播放计划
4. 原生播放器加载 HLS/直链 URL（附加 `?access_token=`）
5. 播放进度定时 `POST /media/:id/progress` 上报，首页「继续观看」读取 `GET /user/history`

### 3.2 技术选型

| 层次 | 技术 | 说明 |
|------|------|------|
| 跨端框架 | **react-native-tvos** | 基于 RN 的电视 fork，支持 Android TV 与 tvOS 单仓构建 |
| 语言 | TypeScript | 与 Web `web/src/api/client.ts`、移动端类型定义对齐复用 |
| 导航 | React Navigation 6+（TV 适配） | Stack + Tab；禁用不适用的手势，启用 TV focus 集成 |
| 焦点 | `react-native-tv-focus-guide` / RN TV 内置 `TVFocusGuideView` | 焦点陷阱、焦点记忆、货架横向滚动 |
| 状态 | Zustand | 认证、播放器、设置、焦点上下文 |
| HTTP | Axios | Base URL、Token 注入、401 统一处理 |
| 视频 | react-native-video（tvOS/Android TV 分支） | ExoPlayer（Android TV）；AVPlayer（tvOS） |
| 音频 | react-native-track-player 或 react-native-video 音频模式 | 专辑连播、Now Playing 元数据（平台能力范围内） |
| 图片 | react-native-fast-image / Image | 缩略图缓存；全屏 Lightbox + 幻灯片 |
| PDF | WebView + pdf.js 或 react-native-pdf | 分页阅读，遥控器翻页 |
| EPUB | WebView + epub.js | 复用 Web/移动端方案 |
| 安全存储 | Android EncryptedSharedPreferences / tvOS Keychain | JWT 加密持久化 |
| 国际化 | i18next | zh-CN、zh-TW、en、ja、ko |
| 构建 | Gradle（Android TV）+ Xcode（tvOS） | 独立 scheme / productFlavor |

### 3.3 平台差异

| 能力 | Android TV | Apple TV (tvOS) |
|------|------------|-----------------|
| 最低系统 | Android 8.0（API 26） | tvOS 15.0 |
| 导航输入 | D-pad、遥控器、部分游戏手柄 | Siri Remote（触摸板 + 方向）、蓝牙键盘 |
| 焦点样式 | 系统 Leanback 缩放/边框 | tvOS 视差放大、光晕焦点环 |
| 返回键 | `Back` / `KEYCODE_BACK` | Menu 键 / 手势返回（无硬件返回） |
| 应用入口 | Leanback Launcher、Banner | tvOS 主屏图标 |
| 推荐位 | Android TV Channels / Watch Next（二期） | Top Shelf Extension（二期） |
| DRM | Widevine L1/L3（设备相关） | FairPlay Streaming |
| 后台音频 | 前台 Activity 播放为主 | 音频可后台；视频 PiP 有限 |
| 商店分发 | sideload APK / Play Store（TV 类别） | App Store tvOS |
| 文本输入 | 屏幕键盘 / 外接键盘 / 语音（系统） | 屏幕键盘 / Remote 输入 / dictation |
| 证书 | 用户自签 HTTPS 需网络配置或用户确认 | ATS 例外需 Info.plist 配置 |

**统一抽象层：**

- `Platform.isTV` / `Platform.OS` 分支处理 DRM、Top Shelf、Leanback 仅 tvOS/Android 特有模块
- 遥控器映射统一为 `RemoteInput` 模块：`Up/Down/Left/Right/Select/PlayPause/Menu/Back`
- 焦点组件 `FocusableCard`、`HorizontalShelf` 跨平台复用，样式按平台微调

## 4. 功能需求

### 4.1 服务器连接与认证

#### 4.1.1 服务器配置

**描述**：首次启动或设置页配置 Vauldy 服务器地址。

| 字段 | 必填 | 说明 |
|------|------|------|
| 服务器地址 | 是 | 如 `http://192.168.1.100:8200` 或 `https://vauldy.home:8200` |
| 连接测试 | — | 调用 `GET /health` 验证可达性 |

**电视端特殊要求：**

- 大字号输入框，焦点在字段间用方向键切换
- 支持外接蓝牙/USB 键盘输入 URL（Android TV / tvOS）
- 屏幕键盘弹出时不丢失焦点上下文
- 连接失败明确提示：超时、证书错误、主机不可达
- 配置持久化（AsyncStorage + 加密敏感项）

**验收标准：**

- 首次启动无配置时进入 `/setup` 引导页
- 地址保存后可修改与清除
- `GET /health` 成功显示绿色状态，失败显示可读错误

#### 4.1.2 用户登录

**描述**：使用用户名密码获取 JWT 会话。

| API | 方法 | 路径 |
|-----|------|------|
| 登录 | POST | `/api/v1/user/login` |
| 用户信息 | GET | `/api/v1/user/info` |
| 登出 | POST | `/api/v1/user/logout` |
| 品牌信息 | GET | `/api/v1/branding` |

**基本流程：**

1. 用户在焦点表单输入用户名、密码（电视屏幕键盘）
2. 客户端调用登录接口，获取 `{ token, expires_in_hours }`
3. Token 加密存入平台安全存储
4. 跳转首页，后台拉取 `user/info` 填充角色与权限
5. 后续 JSON 请求携带 `Authorization: Bearer <token>`
6. 展示 `branding.app_name`、Logo（如有）

**验收标准：**

- 登录成功进入首页；失败显示服务端错误信息
- Token 过期或 401 自动跳转登录页并清除本地会话
- 密码字段支持显示/隐藏切换（焦点可操作）
- 登录页支持 QR 码辅助登录（三期可选：Web 扫码授权）

#### 4.1.3 权限与家长控制

**描述**：遵循服务端 `user/info` 返回的权限字段。

| 字段 | 行为 |
|------|------|
| `can_play` | 为 false 时禁止进入播放器，详情页隐藏播放按钮 |
| `can_download` | 为 false 时隐藏下载相关入口（电视端下载为三期） |
| `library_scope: selected` | 仅展示已授权媒体库 |
| `parental_enabled` | 触发分级/时段限制时弹出 PIN 解锁对话框 |

**PIN 解锁 API**：`POST /api/v1/user/parental/unlock` → `{ unlock_token, expires_in_minutes: 20 }`

**电视端要求：**

- PIN 输入使用大数字键盘 UI，方向键移动焦点
- 解锁失败震动/视觉反馈（平台允许范围内）
- 解锁状态在会话内缓存，过期重新验证

### 4.2 主界面、导航与遥控器

#### 4.2.1 整体布局（Leanback 风格）

**描述**：登录后主界面采用 **左侧主导航 + 右侧内容区** 或 **顶部 Tab + 横向货架**，适配 1920×1080 / 4K 安全区。

**主导航项（建议）：**

| 导航 | 页面 | 说明 |
|------|------|------|
| 首页 | Home | 媒体库入口、继续观看、最近添加 |
| 浏览 | Browse | 按库类型浏览 |
| 搜索 | Search | 全局搜索 |
| 收藏 | Favorites | 收藏媒体列表 |
| 历史 | History | 播放历史 |
| 设置 | Settings | 账户、服务器、语言 |

**布局要求：**

- 导航栏固定宽度，焦点在导航与内容区间可循环切换
- 内容区以 **横向滚动货架（Shelf）** 展示：每行一个主题（继续观看、某库最新、收藏等）
- 卡片最小聚焦尺寸满足 10-foot 可读（标题 ≥ 24sp 等效）
- 焦点卡片放大 1.08× + 品牌色边框（`#00a4dc`）
- 滚动时保持焦点项可见（`scrollToFocusedChild`）

#### 4.2.2 焦点导航规范

| 规则 | 说明 |
|------|------|
| 初始焦点 | 每页进入时焦点落在逻辑首项（如首页第一个货架首卡） |
| 方向键 | ↑↓ 切换货架/列表行；←→ 同行内切换卡片 |
| 确认 Select | 进入详情或立即播放（可配置「单击播放」vs「进入详情」） |
| Menu / 长按 | 上下文菜单：收藏、加入播放列表、标记已看 |
| 焦点记忆 | 从详情返回列表时恢复离开前的卡片焦点 |
| 焦点陷阱 | 模态层（PIN、设置弹窗）内焦点不逃逸 |
| 不可聚焦 | 装饰性元素 `focusable={false}`，减少焦点路径长度 |

#### 4.2.3 遥控器与媒体键映射

| 按键 | Android TV | Apple TV | 功能 |
|------|------------|----------|------|
| 方向键 | D-pad | Siri Remote 滑动/边缘 | 移动焦点 |
| 确认 | Enter / Center | Click | 选中 |
| 返回 | Back | Menu（或手势） | 返回上一级 |
| 播放/暂停 | Media Play/Pause | Play/Pause | 播放器内切换 |
| 快退/快进 | Media RW/FF（如有） | 播放器 UI | ±10s seek |
| Home | 系统 | 系统 | 不拦截，交还系统 |

**播放器内优先捕获媒体键**，避免事件冒泡至导航层。

#### 4.2.4 首页

**展示内容：**

- 已授权 **媒体库** 大卡（名称、类型、数量、封面渐变）
- **继续观看/收听**（`GET /api/v1/user/history`）
- **最近添加**（`GET /api/v1/media?sort=created_desc&limit=24`，按库分组）
- 快捷货架：收藏精选、播放列表入口（二期）

**验收标准：**

- 进入首页 ≤ 2s 展示骨架屏后填充数据（局域网）
- 空库友好提示与引导浏览
- 焦点在货架间纵向切换流畅，无焦点丢失

### 4.3 媒体库浏览

#### 4.3.1 库列表

**API**：`GET /api/v1/library`

**库类型与浏览模式映射：**

| library.type | 浏览 UI |
|--------------|---------|
| movie / video | 海报网格货架 / 列表 |
| tv / anime | 剧集列表 → 季/集详情页 |
| music | 专辑 / 艺术家 / 流派 / 曲目 Tab 货架 |
| photo | 时间线 / 分类 / 地点 / 人物 |
| document | 文件夹树 / 作者 / 格式 / 标签 / 年份 |

#### 4.3.2 通用媒体列表

**API**：`GET /api/v1/media`

| 参数 | 说明 |
|------|------|
| `library_id` | 库 ID |
| `file_type` | video / audio / image / document |
| `q` | 全文模糊搜索 |
| `sort` | id_desc / created_desc / taken_desc |
| `limit` | 分页上限 |
| `photo_tag` / `photo_place` / `photo_person` | 照片筛选 |

**电视端展示要求：**

- 默认 **网格货架**（每行 5–8 张海报，4K 下可 8–10）
- 海报懒加载，`/media/:id/poster.jpg?access_token=`
- 向下滚动加载更多（无限滚动 + 焦点不跳动）
- 显示标题、年份、时长（视频）、艺术家（音乐）
- 列表/网格视图切换（设置项，二期）

#### 4.3.3 影视剧集

| API | 用途 |
|-----|------|
| `GET /library/:id/series` | 剧集列表 |
| `GET /series/:id` | 剧集详情 |
| `GET /series/:id/play-target` | 续播目标 `{ media_id, position }` |
| `GET /season/:id/episodes` | 季集列表 |

**剧集详情页（电视优化）：**

- 左侧/背景大图（模糊背景 + 前景海报）
- 季选择为横向 Tab 货架
- 集列表每行显示集号、标题、时长、观看进度条
- 焦点在集上按确认：**直接播放** 并恢复进度
- 「从头播放」/「播放下一集」快捷按钮

**验收标准：**

- 季/集结构与服务端一致
- 续播集标记「继续」徽章
- 播放完一集后提示「下一集」（10 秒倒计时，可取消）

#### 4.3.4 音乐库

### 4.6 视频播放

#### 4.6.1 播放流程

```
用户按确认播放
    │
    ▼
GET /api/v1/media/:id/hls  ──► 播放计划 JSON
    │
    ├── mode: direct      ──► GET /media/:id/play（渐进式 MP4）
    ├── mode: hls         ──► HLS master URL
    ├── mode: hls_drm     ──► HLS + Widevine/FairPlay
    ├── mode: hls_powerdrm──► HLS + PowerDRM key
    ├── mode: hls_aes_128 ──► HLS + AES-128 key
    └── JIT session       ──► /jit/session/:id/master.m3u8
    │
    ▼
原生播放器全屏加载（URL 附加 ?access_token=）
    │
    ▼
POST /media/:id/playback/start
POST /media/:id/progress（定时上报）
POST /media/:id/playback/end
```

#### 4.6.2 播放器 UI（电视）

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 播放/暂停 | P0 | 遥控器 Play/Pause 或确认键 |
| 进度条 Seek | P0 | 左右键微调，长按快进/快退 |
| 全屏 | P0 | 默认全屏，无小窗 |
| 字幕切换 | P1 | 加载 `/subtitles/:sid/vtt` |
| 多音轨 | P2 | HLS `/atrack/:stream/` |
| 倍速 | P2 | 0.75x ~ 1.5x（电视较少用，二期） |
| 预览缩略图 | P2 | sprite.jpg + thumbs.vtt |
| 下一集 | P1 | 剧集自动连播提示 |
| DRM | P1 | 按平台 Widevine / FairPlay |

**控件交互：**

- 播放时 UI 隐藏；任意方向键或确认 **唤起控制条**（5s 无操作隐藏）
- 控制条获焦：进度条、字幕、音轨、返回
- 返回 / Menu：退出播放，上报进度

#### 4.6.3 DRM 平台适配

| DRM 类型 | Android TV | Apple TV |
|----------|------------|----------|
| Widevine | ExoPlayer + `/drm/widevine/license` | — |
| FairPlay | — | AVPlayer + `/drm/fairplay/cert` + `/drm/fairplay/license` |
| PowerDRM | 自定义 key + `/drm/powerdrm/key` | 同左或降级提示 |
| HLS AES-128 | `/drm/hls/aes128/key` | 同左 |

**一期策略：**

- 优先支持 **非 DRM 库**、**HLS 预转码**、**直链** 播放
- DRM 库按平台能力适配；不支持时提示「该内容需在其他端观看」

#### 4.6.4 JIT 实时转码

- 播放计划含 `session_id` 时使用 `/api/v1/jit/session/:sessionID/master.m3u8`
- `POST pause / resume / seek / end` 会话控制
- 首缓冲 Loading 全屏提示，超时 30s 可重试

**验收标准：**

- 非 DRM 视频 HLS/直链正常起播
- 断点续播：读取服务端/本地进度 seek
- 退出时进度准确上报，首页「继续观看」更新

## 5. 接口集成规范

### 5.1 认证方式

| 场景 | 认证方式 |
|------|----------|
| JSON API 请求 | Header: `Authorization: Bearer <JWT>` |
| 流媒体 / 图片 / HLS 分片 | `?access_token=<JWT>`（播放器、Image 无法设 Header） |
| 健康检查 | 无需认证 |

### 5.2 URL 构建规则

与 Web 端对齐，复用 `withAccessToken`、`mediaPosterSrc`、`mediaPlaySrc` 逻辑（`web/src/api/client.ts`）。

```typescript
const apiBase = `${serverUrl}/api/v1`;

function assetUrl(path: string, token: string): string {
  const sep = path.includes('?') ? '&' : '?';
  return `${serverUrl}${path}${sep}access_token=${encodeURIComponent(token)}`;
}

// 示例
// 海报: /api/v1/media/123/poster.jpg?access_token=...
// 播放: /api/v1/media/123/play?access_token=...
// HLS:  /api/v1/media/123/hls/master.m3u8?access_token=...
```

### 5.3 请求头

| Header | 值 |
|--------|-----|
| Authorization | Bearer \<JWT\>（JSON API） |
| Content-Type | application/json |
| Accept | application/json |
| User-Agent | Vauldy-TV/1.0 (Android TV \| tvOS) |

### 5.4 错误处理

| HTTP 状态 | 客户端行为 |
|-----------|------------|
| 401 | 清除 Token，跳转登录页 |
| 403 | 全屏提示权限不足；家长控制触发 PIN 流程 |
| 404 | 媒体不存在或已删除 |
| 5xx | 服务端错误，支持重试（焦点在「重试」按钮） |
| 网络不可达 | Toast/横幅 + 离线图标 |

### 5.5 请求超时

| 类型 | 超时 |
|------|------|
| 普通 API | 30 秒 |
| 流媒体 | 120 秒 |
| 健康检查 | 5 秒 |
| JIT 首片 | 60 秒 |

### 5.6 分页与性能

- 列表默认 `limit=48`（电视一屏较少，但横向货架需预加载）
- 图片请求失败降级占位图
- 同一货架并发图片请求 ≤ 6

## 6. 非功能需求

### 6.1 性能

| 指标 | 目标 |
|------|------|
| 冷启动 | ≤ 4 秒（已登录，中端电视盒子） |
| 首页首屏 | ≤ 2.5 秒（局域网） |
| 焦点切换 | ≤ 100ms 视觉响应 |
| 货架滚动 | 60fps（1080p）；4K 设备 ≥ 30fps |
| 缩略图 | 渐进显示，占位图兜底 |
| 视频起播 | ≤ 5 秒（预转码 HLS）；JIT 首片 ≤ 20 秒 |
| 内存 | 浏览态 ≤ 200 MB；播放态 ≤ 450 MB |
| 安装包体积 | ≤ 50 MB（因平台而异） |

### 6.2 兼容性

| 平台 | 最低版本 | 测试参考设备 |
|------|----------|--------------|
| Android TV | API 26（Android 8.0） | 小米盒子、NVIDIA Shield、Google TV |
| Apple TV | tvOS 15.0 | Apple TV 4K (2nd gen+)、Apple TV HD |

**分辨率：**

- 基准布局 1920×1080（1080p）
- 4K 电视安全缩放，图片资源优先 vector / 多密度
-  overscan 安全区：上下左右各 5% 边距

### 6.3 安全

- JWT 存储于 EncryptedSharedPreferences（Android）/ Keychain（tvOS）
- 支持 HTTPS；自签名证书需用户确认或网络安全配置（Android `network_security_config`）
- 不在日志中输出 Token、密码
- 禁止 WebView 加载非配置域名的任意 URL（CSP 等价策略）
- 登出清除 Token 与内存缓存

### 6.4 网络

- 优先局域网；支持远程（端口转发 / VPN / Tailscale 等）
- 弱网下降级：照片 thumb、视频较低码率 HLS variant
- 无网络时全屏离线提示，已加载元数据只读展示

### 6.5 可用性与无障碍

- **焦点导航** 为一级能力：所有可交互元素可被遥控器到达
- 焦点环对比度 ≥ 3:1，品牌色 `#00a4dc` 高亮
- 深色主题默认，背景 `#0f1419`
- 支持 **动态字体** 有限缩放（系统设置跟随）
- TalkBack / VoiceOver 基础朗读（标签 `accessibilityLabel`）
- 减少动画选项（尊重系统「减少动态效果」）

### 9.2 验收场景（核心）

1. 首次启动配置服务器 → 登录 → 首页展示媒体库与继续观看
2. 浏览电影库 → 详情 → 全屏播放 → 退出后续播正确
3. 浏览剧集 → 选季选集 → 播放 → 下一集提示
4. 浏览音乐库 → 播放专辑 → 迷你条 → 全屏歌词（二期）
5. 浏览照片 → 全屏 → 幻灯片模式
6. 浏览文档 → PDF 阅读 → 退出恢复进度
7. 搜索关键词 → 结果播放
8. 收藏/取消收藏，收藏页可见
9. 401 Token 过期自动跳转登录
10. 403 受限内容提示，家长 PIN 解锁后可播放
11. Android TV 与 Apple TV 双平台焦点导航完整可用
12. DRM 内容在对应平台可播放（二期）

### 9.3 性能与稳定性

- 连续播放 2 小时无内存泄漏（增长 < 20%）
- 首页快速切换导航 50 次无崩溃
- 弱网（300ms 延迟）下缩略图降级正常

### 10.1 后端 API 路由索引

> 完整路由定义见主项目 `Vauldy/api/router.go`

| 分组 | 鉴权 | 用途 |
|------|------|------|
| Public | 无 | login, branding, health |
| auth | Bearer | 浏览、收藏、设置等 JSON API |
| play | Bearer 或 access_token | 流媒体、图片、HLS、DRM |
| adm | Bearer + Admin | 管理功能（电视端不涉及） |

**电视端常用端点速查：**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 健康检查 |
| POST | `/user/login` | 登录 |
| GET | `/user/info` | 用户信息 |
| GET | `/library` | 媒体库列表 |
| GET | `/media` | 媒体列表/搜索 |
| GET | `/media/:id` | 媒体详情 |
| GET | `/media/:id/hls` | 播放计划 |
| GET | `/media/:id/play` | 直链流 |
| POST | `/media/:id/progress` | 播放进度 |
| GET | `/user/history` | 继续观看 |
| GET | `/favorites` | 收藏列表 |
| GET | `/branding` | 品牌信息 |
